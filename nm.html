About the course

This course is scheduled to be taught Spring ’15 at SHC. This is a 400-level course for math majors with a pretty standard catalogue description. The class requires Calc 3 and an Intro to OOP Programming in C++. Numerical methods in C with particular emphasis on linear algebra and differential equations is the classic choice. Often other topics such as the approximation of functions is introduced alongside functional analysis in more advanced courses. Since neither linear or DEQ are assumed I am attempting to stay away from these. I want to focus more on root finding, interpolation, integration and differentiation. I would like to compare the numerical (approximation, e.g. NumPy) algorithms to symbolic (exact, e.g. SymPy) algorithms. Further, I want students to implement these algorithms and compare run-times.

So, why Ruby? C/C++ is the classic choice and the students have some experience with it. Indeed if I were to implement this for use in a large scale professional setting, this would be the way to go, but we are not. I anticipate at least a few seniors who haven't programmed since the OOP course their freshman year. They will be very rusty. I want to provide short clean looking code which handles all the technical bits and the students will simply write in the algorithm as a function. This calls for a higher level language. Python is the next obvious choice. Python has all the algorithms implemented in standard libraries and would at first seem very desirable. As an instructor, I don't want to deal with the 2.x/3.x mess. It isn't really a mess, if you know a which version you are coding, one can easily hop around the internet and implement what you find. But this isn't a "how to program" course and these students will have no experience with python and the 2.x/3.x would show up right away with "Hello world". Also python and it's libraries are a little hard to install. Ruby is easy to install. I really like the RailsInstaller. Ruby is easy to read/write. Ruby is quite trendy at the moment. Also because Ruby really isn't used for numerical stuff that often, they will have trouble googling the answer. I like when a program reads a file and produces an output. GNUPlot works out of the box and easily provides all the graphical support for a nice output png.
